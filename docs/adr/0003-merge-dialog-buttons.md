# 0003 合併 Likes Dialog 內的清理按鈕

## 背景

likes dialog 內原本同時有「殺螂囉」與「定點絕」兩顆按鈕。beta40 之後，「殺螂囉」已經變成「定點絕」的子集：兩者都會處理目前 dialog 內的帳號，而「定點絕」還會把貼文加入水庫供後續追蹤。

兩顆按鈕並排時，使用者需要理解「只清這批」與「清這批並持續追」的差異，但實際主流程已經偏向貼文水庫。保留兩顆按鈕會讓 UI 入口顯得重複，也增加錯選機率。

## 選項

- (a) 拿掉「殺螂囉」，讓「定點絕」成為唯一入口，且永遠加入水庫。
- (b) 保留「殺螂囉」但改名為「立即清這批」，「定點絕」改名為「立即清+持續追」。
- (c) 兩顆都保留，但加 tooltip 區分用途。

## 決定

採用 (a)：likes dialog 內兩顆按鈕合一，保留「定點絕」功能與位置，移除「殺螂囉」。

理由是目前「殺螂囉」已經是「定點絕」的子集，繼續保留會放大 UI 混淆。「只想清一次不追」屬於較邊角的使用情境，可透過水庫 modal 手動按 `×` 移除該 entry。

## 後果

好處是 dialog 操作入口更單純，使用者不需要在兩個高度相似的按鈕間做選擇。

副作用是每次清理都會新增 `POST_QUEUE` entry，貼文水庫可能變得比較雜。

未來可以加入「跑完自動移除」選項，讓只想清一次的使用者不必手動整理水庫。
