# 平台呈現貼文連結／話題風險備忘

日期：2026-04-20
狀態：討論中

## 目的

整理 ThreadsBlocker 平台若要呈現「哪些貼文被大量封鎖」「哪些話題可能存在異常擴散／協同行為」時的主要法規與產品風險，並提出可執行的降風險設計。

這份文件是產品備忘，不是正式法律意見。

## 目前判斷

### 1. 只顯示貼文連結，比直接顯示陌生人 ID 風險低

- 單純外連到原平台貼文，通常比直接顯示帳號 ID、集中列名單、重貼全文或截圖更安全。
- 但只要連結可穩定回到特定自然人的帳號或貼文，仍可能被視為可間接識別個人的資料，個資風險只是下降，不是消失。

### 2. 最大風險不在「連結」，而在「標籤」

- 如果平台對單篇貼文或話題下了負面判斷，例如：
  - `帶風向`
  - `網軍文`
  - `操弄輿論`
- 風險會從單純資料整理，升級成名譽、信用、隱私與不實陳述爭議。

### 3. 純連結通常比重貼內容安全

- `純連結`：風險相對低。
- `自家快取全文 / 截圖 / 縮圖 / snippet / 鏡像頁`：會額外碰到重製、公開傳輸與平台條款風險。

## 風險分級

### 低風險

- 只做話題層級統計，不顯示單篇貼文。
- 只顯示聚合結果，例如：
  - `近 7 日封鎖量上升話題`
  - `本平台樣本中高封鎖率話題`

### 中低風險

- 顯示單篇貼文連結，但只做中性、描述性標示，例如：
  - `樣本內封鎖率偏高`
  - `短時間內被多位使用者標記為不想再看到`
  - `異常擴散訊號`

### 中高風險

- 顯示單篇貼文連結，並以模型或規則推估「可能存在協同行為」。
- 這條路可以做，但必須滿足：
  - 只寫 `可能 / 訊號 / 指標`
  - 提供方法說明
  - 明示 `非事實認定`
  - 提供申訴或更正機制

### 高風險

- 顯示單篇貼文連結，直接標成：
  - `帶風向`
  - `網軍`
  - `假帳號操作`
  - `操弄輿論`
- 這類語言太接近對特定作者或內容作負面定性，名譽與誹謗風險明顯升高。

## 如果仍想呈現「類似狀態」，建議怎麼做

### 方案 A：只做話題，不點名單篇貼文

做法：

- 首頁只顯示話題、關鍵詞、群聚主題。
- 顯示：
  - 封鎖率
  - 時間趨勢
  - 近 24 小時增幅
  - 參與樣本數

適合用途：

- 戰情室
- 趨勢觀察
- 外部公開頁

優點：

- 風險最低
- 產品敘事清楚
- 不直接把壓力集中到單一作者或貼文

### 方案 B：貼文可見，但預設只顯示中性指標

做法：

- 顯示單篇貼文連結。
- 不顯示作者帳號作為主標。
- 只顯示中性欄位，例如：
  - `樣本內封鎖率`
  - `封鎖數變化`
  - `短時間內重複出現次數`
  - `互動異常密度`

建議文案：

- `此連結在本平台樣本中出現異常擴散訊號`
- `此連結在短時間內被多位使用者封鎖`
- `此連結與近期高封鎖話題高度相關`

不要使用：

- `這篇在帶風向`
- `這是網軍文`
- `作者疑似操控輿論`

### 方案 C：先顯示話題，單篇連結放第二層

做法：

- 首頁只顯示話題卡片。
- 點進去後才看到相關連結清單。
- 清單排序依封鎖量、時間密度或異常分數。

額外保護：

- 設最小樣本門檻，例如至少 10 位以上獨立使用者、或至少 24 小時內達一定量才顯示。
- 低樣本數只顯示話題，不顯示單篇。

### 方案 D：單篇可見，但只對登入使用者開放

做法：

- 公開頁只顯示話題與聚合數據。
- 單篇貼文連結、指標詳情只對登入用戶開放。
- 再搭配：
  - `noindex`
  - 禁止搜尋引擎索引
  - 不提供公開 API
  - 不開放大量匯出

這不能完全消除風險，但能避免平台變成可被搜尋、轉載與二次擴散的「指控索引」。

### 方案 E：保留連結，但不直接下判斷

做法：

- 只顯示：
  - `高封鎖率`
  - `高重複出現率`
  - `擴散速度異常`
- 把判讀權交給使用者。

這是目前最穩的平衡點：

- 有情報價值
- 不直接替平台承擔「事實認定者」角色

## 產品文案紅線

### 建議使用

- `異常擴散訊號`
- `高封鎖率`
- `樣本中重複出現`
- `短時間密集傳播`
- `可能存在協同行為`
- `非事實認定，僅反映本平台樣本訊號`

### 建議避免

- `帶風向`
- `操弄`
- `網軍`
- `洗版`
- `假訊息來源`
- `惡意作者`

除非有非常明確、可驗證且可抗辯的事實基礎，否則不應把這些詞放進 UI 主文案。

## 最低限度的降風險護欄

如果要呈現單篇貼文連結，至少應同時滿足下列條件：

1. 只做外連，不快取原文、不鏡像、不存截圖。
2. 只顯示聚合訊號，不顯示誰封鎖了它。
3. 設定最小樣本門檻，避免少量使用者就把單篇內容推上榜。
4. 使用中性、描述性語言，不做主觀定罪。
5. 對外加上方法說明與免責文字：
   - `本頁資訊反映平台樣本中的封鎖與擴散訊號，不代表對貼文內容真實性、動機或作者身分之事實認定。`
6. 提供申訴、移除、補充說明或人工覆核入口。
7. 不開放搜尋引擎索引、不提供公開批次匯出。
8. 若資料來源依賴自動化抓取，需額外評估 Meta / Instagram / Threads 平台條款風險。

## 建議的產品落地順序

### 第一階段

- 只做話題頁。
- 只顯示聚合統計與時間趨勢。

### 第二階段

- 允許登入用戶查看單篇貼文連結。
- 僅顯示中性指標，不顯示負面定性。

### 第三階段

- 若真的需要 `可能存在協同行為` 類型訊號：
  - 必須先完成方法揭露
  - 建立申訴入口
  - 建立人工覆核流程

## 暫定結論

可以做，但應優先採用下面這個版本：

- `公開頁 = 話題 + 聚合統計`
- `登入後 = 單篇連結 + 中性訊號`
- `全站避免使用帶風向、網軍、操弄等定性用語`

這個版本能保留產品價值，同時把個資、名譽、著作權與平台條款風險壓到相對可控的範圍。

## 目前採用方向（V1）

目前先往「公開頁只呈現話題與聚合訊號」的版本實作，不以單篇貼文、作者 ID 或原文連結作為首頁重點。

### V1 公開頁要呈現的內容

- 話題名稱或話題群組
- 聚合統計
- 近 7 日封鎖率
- 24 小時增幅
- 樣本數
- 中性 badge：
  - `高封鎖率`
  - `短時間密集傳播`
- 方法說明
- 免責文字

### V1 先不做的內容

- 單篇貼文連結
- 完整 Threads URL
- 使用者 ID / `@username`
- 原文摘錄
- 截圖或縮圖
- 誰封鎖了它
- 政治或動機性標籤

### V1 呈現原則

- 平台只陳述可量化、可重算、可說明的方法事實。
- 政治或動機性標籤留給使用者自行判讀。
- 首頁優先做話題戰情與群體趨勢，不做單一來源點名。
